Projet SAE - Trello Trollé

Contexte du Projet

Le projet SAE Trello Trollé a été réalisé durant le Semestre 4 du BUT Informatique. Ce projet avait pour but d'améliorer une application inspirée de Trello, mais volontairement mal codée, afin de renforcer nos compétences en bonnes pratiques de développement et en optimisation de code.

Nous avions deux objectifs principaux :

  • Revoir et corriger les mauvaises pratiques de code (PHP et CSS).
  • Rendre l'interface dynamique en JavaScript natif.

Méthodes de Travail et Résultats

Le projet a été organisé en deux grandes phases :

  • Phase 1 : Révision des bonnes pratiques
    • Refactorisation du code PHP pour respecter les standards (PSR).
    • Nettoyage et optimisation du CSS.
    • Mise en place d'une structure de projet plus claire.
  • Phase 2 : Dynamisation de l'interface
    • Ajout de fonctionnalités dynamiques en JavaScript natif (glisser-déposer des cartes, mises à jour en temps réel des colonnes).
    • Communication asynchrone entre le frontend et le backend avec AJAX.

Résultats obtenus

  • Une interface utilisateur fluide et interactive.
  • Une amélioration significative des performances et de la maintenabilité du code.
  • Un backend robuste et conforme aux standards de développement modernes.

Preuves Visuelles

Technologies Utilisées

  • PHP pour la gestion du backend.
  • CSS pour la mise en page et le style.
  • JavaScript natif pour l'interactivité de l'interface.
  • Docker pour le déploiement de services comme la base de données.

Compétences Mobilisées

  • C1 : Réaliser un développement d'application
    • Refactorisation et correction des bugs dans le code PHP.
    • Ajout de nouvelles fonctionnalités côté serveur et client.
  • C2 : Concevoir des interfaces utilisateur
    • Création d'une interface intuitive avec CSS et JavaScript natif.
    • Amélioration de l'ergonomie et de la fluidité de navigation.
  • C3 : Gérer un projet
    • Gestion des tâches en équipe et respect des délais.
    • Suivi de l'avancement sur Trello.
  • C4 : Gérer des données de l'information
    • Optimisation des requêtes SQL pour améliorer les performances.
    • Mise en place d'une architecture de base de données efficace.
  • C5 : Collaborer au sein d'une équipe informatique
    • Utilisation de Git pour le versionnage collaboratif.
    • Revues de code régulières pour garantir la qualité.
  • C6 : Assurer la qualité et la sécurité des services informatiques
    • Validation des entrées utilisateur pour éviter les failles XSS et SQL Injection.
    • Amélioration de la maintenabilité et de la lisibilité du code.
