Projet API REST - Gestion d'Événements Musicaux

Contexte du Projet

Ce projet a été réalisé dans le cadre du TD4 de Symfony avec API Platform. L'objectif principal était de développer une API REST permettant de gérer des événements musicaux. Chaque événement peut inclure plusieurs scènes, qui elles-mêmes sont composées de plusieurs parties de concert, chaque partie étant attribuée à un artiste.

Méthodes de Travail

Nous avons suivi une méthodologie agile pour la gestion du projet avec des livrables réguliers. Voici les outils et technologies utilisés :

  • Symfony pour le backend et la structure de l'API.
  • API Platform pour la création et la gestion des routes RESTful.
  • GitHub pour le versionnement et la collaboration.

Résultats Obtenus

  • Gestion complète des événements musicaux, incluant les scènes et les parties de concert.
  • Authentification des utilisateurs via JWT.
  • Possibilité pour les organisateurs de créer, modifier et supprimer des événements.
  • Inscription des participants avec gestion des conflits d'horaires.
  • Routes pour consulter les événements publics et leurs détails.
  • Fixtures pour peupler la base de données
